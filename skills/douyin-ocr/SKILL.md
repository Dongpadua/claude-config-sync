# 抖音内容文字识别（音视频双通道）

当用户提供一条抖音链接并要求提取文字内容时使用。

## 核心理念

- **OCR 为主，音频为辅**：RapidOCR 画面字幕是权威来源（人能看到的东西），Whisper 音频仅用于补充 OCR 漏掉的人声
- **禁止编辑**：OCR 识别到什么就输出什么，不加字、不漏字、不改字。你不是作者，无权润色
- **OCR 模糊时原样保留**：形近字/模糊字原样输出，不要脑补推断。判断是用户的事
- **不添加分析**：默认只输出正文文字。只有用户明确要求时才加分析和对比表格
- **OCR 碎片拼接连成段落**：字幕拆成单句是正常的，按时间线拼接即可
- **音频仅做兜底**：只有 OCR 完全没扫到的内容，才用音频补上
- **画面噪音分类**：顶部区域 = 标题/标签/互动信息（选择性保留），底部区域 = 字幕（保留）

## 错误教训（2026-06-15）

两次关键错误，必须避免：

1. **人工干预错误**：OCR 识别到 `由裹`（"由衷"的形近），我脑补推断为"郑重"而非"由衷"。不要替作者选词，确有歧义时列出可能性让用户判断。
2. **手工漏字**：整理文字时漏掉了"一定"后面的"是"。直接复制拼接，不要凭记忆打字。
3. **原则**：能直接复制的就不要手动输入。每多做一个人工步骤就多一个出错机会。

## 执行流程

### 1. 下载 + 音频转写

复用 `mrcarlsama-social-transcriber` 的下载和 Whisper 转写链路：

```bash
SKILL_DIR="$HOME/.claude/skills/mrcarlsama-social-transcriber"
uv run --script "$SKILL_DIR/scripts/bootstrap.py" --ensure
uv run --script "$SKILL_DIR/scripts/run_one.py" "<抖音链接>"
```

### 2. 视频 OCR

```bash
SKILL_DIR="$HOME/.claude/skills/douyin-ocr"
python "$SKILL_DIR/scripts/ocr_video.py" "<视频文件路径>" "<输出json路径>"
```

OCR 配置：
- 引擎：RapidOCR (ONNX Runtime)
- 扫描区域：顶部 30%（标题/标签）+ 底部 40%（字幕）
- 帧差检测：均值差 < 12 则跳过重复帧
- 分辨率：50% 压缩加速

### 3. 双通道合并

用模型能力做，不要写脚本自动合并。人工（模型）对比两个通道的差异，规则：

- 音频和 OCR 一致 → 直接采用
- 只有音频有 → 采用音频，标记 `[仅音频]`
- 只有 OCR 有 → 采用 OCR，标记 `[仅画面]`
- 两者不一致 → 以 OCR 画面字幕为准（人能看到的字比 AI 听的要准），但保留标注
- OCR 碎句 → 按时间线拼接为连续段落
- 顶部画面文字 → 区分：标题保留，互动信息/账号名等包装元素丢弃

### 4. 输出

默认只输出校对后的纯文本正文。格式：

```
正文内容第一行
第二行
...
```

只有在以下情况才额外输出：
- 用户要求分析 → 加对比表
- 用户要求 Word → 生成 Word 文档
- 确实有关键歧义 → 用 `[音频: xxx / OCR: xxx]` 标注让用户判断

## 注意事项

- 猫meme/图文类视频字幕在底部，知识类视频标题在顶部——根据视频类型调整扫描重点
- RapidOCR 首次运行会下载 ONNX 模型（~50MB），之后秒启
- 保持 `_ocr_result.json` 供复查，不要删
